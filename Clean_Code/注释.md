# 注释

## 注释的目的

* 提高晦涩难懂的代码的可读性
* 隐藏代码复杂细节
* 弥补在用代码表达意图时遭遇的失败

## 注释的局限性

* 注释存在的时间越久，离其所描述的代码就越远
* 注释不能美化糟糕的代码

## 写注释的原则

* 优先考虑能不能通过改变代码来避免写注释
* 不准确的注释不如没有注释
* 解释代码为什么这么做而不是怎么做

## 好的注释

* 法律信息
* 提供信息的注释
* 阐释某些晦涩难明的参数或返回值的意义
* 警示会出现某种后果的注释
* TODO注释
* 放大某种看来不合理之物的重要性（？）
* 公共API接口

## 坏注释

* 显而易见的注释
* 误导性的注释
* 循规式注释
* 能用函数或变量时就别用注释
* 位置标记（？）
* 括号后面的注释
* 归属和署名
* 注释的代码
* 代码与注释没有明显的联系
